# 🏗️ Kafka & Drools Fusion 연동 실시간 FDS 파이프라인 시뮬레이터

Apache Kafka가 실시간 데이터 스트림을 안정적으로 수집·전달하고, Drools Fusion(CEP)이 인메모리 상에서 시간 기반 규칙을 실시간으로 평가하는 고성능·저지연 이상거래 탐지 및 보안 모니터링 시스템을 모사한 Python/Streamlit 프로젝트입니다.

## 🛠️ 실무 구현 3대 핵심 포인트
1. **이벤트 시간(Event Time) 기반 의사 시계(PseudoClock)**: 네트워크 지연 왜곡을 방지하기 위해 서버의 현재 시각 대신 메시지 고유의 타임스탬프(`eventTime`)를 기반으로 엔진 시계를 전진시켜 정확한 패턴을 감지합니다.
2. **컨슈머 스케일 아웃과 샤딩(Sharding)**: 인메모리(RAM) 세션 상태 유지를 위해 `userId`를 Kafka 파티션 키로 지정하여 동일 고객 데이터가 항상 지정된 컨슈머 인스턴스로만 매핑되도록 보장합니다.
3. **규칙(Rule)의 동적 변경**: 애플리케이션 재배포 및 서버 중단 없이 슬라이딩 윈도우 범위 및 위험 탐지 임계치를 실시간으로 새로고침하여 규칙에 즉시 반영합니다.

## 📂 프로젝트 디렉토리 구조
```text
├── app.py              # 프론트엔드 UI 렌더링 및 컨트롤러 동작 전담 메인 파일
└── util/
    ├── __init__.py
    ├── kafka_util.py    # 가상 데이터 생성, 파티션 키 추출, 알림 발행 비즈니스 함수
    └── cep_engine.py    # PseudoClock, Stateful Session 및 슬라이딩 윈도우 평가 엔진
```

## ⚡ 설치 및 실행 방법

### 1. 의존성 패키지 설치
본 프로젝트는 웹 대시보드 렌더링을 위해 `Streamlit`, 데이터 구조 조작을 위해 `Pandas` 라이브러리를 필요로 합니다.
```bash
pip install streamlit pandas
```

### 2. 애플리케이션 실행
터미널에서 프로젝트 루트 디렉토리로 이동한 뒤 아래 명령어를 수행합니다.
```bash
streamlit run app.py
```

## 🕹️ 시뮬레이터 사용 가이드
1. **Dynamic Rule Configuration (사이드바)**: 실시간 가동 중에도 슬라이딩 윈도우 범위(초) 및 위험 결제 임계 횟수를 상시 조정할 수 있습니다. 변경 값은 즉시 컨슈머 엔진에 적용됩니다.
2. **가상데이터 발생 ⚡**: 버튼을 누르면 0.5초 간격으로 무작위 사용자 및 국가 코드가 래핑된 가상 결제 이벤트를 생성하여 실시간 파이프라인 연산을 가동합니다.
3. **가상데이터 발생 중지 🛑**: 실시간으로 유동치던 모든 데이터 수집 스트림과 인메모리 세션 상태 테이블을 그 자리에 즉시 동결하여 탐지된 이상거래 로그 분석을 용이하게 합니다.

## 🚨 주요 이상거래 탐지 규칙
* **규칙 1 (타국 결제 발견)**: 슬라이딩 윈도우 설정 시간(예: 10분) 내에 동일한 Partition Key를 가진 사용자가 2개국 이상의 국가(예: KR 결제 후 곧바로 US 결제)에서 거래 시 즉시 차단 API 호출 피드를 발행합니다.
* **규칙 2 (단시간 과다 결제)**: 슬라이딩 윈도우 내에 한 유저가 임계 설정값 이상의 결제를 시도할 경우 비정상 매크로 혹은 탈취 패턴으로 간주하고 경고를 트리거합니다.
